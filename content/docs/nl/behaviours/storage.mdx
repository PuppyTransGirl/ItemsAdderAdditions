Verandert elk aangepast blok of meubel in een opslagcontainer. Er zijn drie modi beschikbaar.

| Type       | Gedrag                                                                          |
|------------|---------------------------------------------------------------------------------|
| `STORAGE`  | Gedeelde container - alle spelers zien dezelfde inventaris                      |
| `SHULKER`  | Inhoud reist mee in het item bij afbreken en wordt hersteld bij herplaatsen     |
| `DISPOSAL` | Prullenbak - inhoud wordt stilzwijgend verwijderd wanneer de GUI wordt gesloten |

Gebruik voor `name` een waarde uit [Sounds.json](https://minecraft.wiki/w/Sounds.json/Java_Edition_values) of een aangepast geluid met `namespace:value`.

Ondersteunt [MiniMessage](https://docs.papermc.io/adventure/minimessage/format/)-opmaak, [PlaceholderAPI](https://modrinth.com/plugin/placeholderapi)-placeholders en [ItemsAdder](https://itemsadder.com/)-lettertypeafbeeldingen.

## Inventaristype

Standaard opent opslag een kist-achtig inventaris waarvan de hoogte wordt bepaald door `rows` (1–6, standaard: 3).
Stel `inventory_type` in om een getypeerd inventaris te openen — `rows` wordt genegeerd als dit is ingesteld.

| Waarde          | GUI-vorm                                    |
|-----------------|---------------------------------------------|
| `furnace`       | Oven (invoer + brandstof + uitvoer)         |
| `blast_furnace` | Hoogoven                                    |
| `smoker`        | Roker                                       |
| `brewing_stand` | Brouwstandaard (3 flesjes + 2 vakjes)       |
| `dispenser`     | Verdeler (3×3)                              |
| `dropper`       | Dropper (3×3)                               |
| `hopper`        | Trechter (1×5)                              |

```yaml
behaviours:
  storage:
    type: STORAGE
    inventory_type: furnace
    title: "<gray>Aangepaste Oven</gray>"
```

## Open variant

Je kunt optioneel `open_variant` instellen om het aangepaste blok, meubel of complexe meubel te laten veranderen naar een andere variant terwijl ten minste een speler de opslag open heeft.

Dit is handig voor dingen zoals:

- kisten die visueel openen
- vaten met een geopend model
- containers met een andere actieve toestand

```yaml
behaviours:
  storage:
    type: STORAGE
    open_variant: my_open_storage
```

`open_variant` moet de ID zijn van een ander aangepast blok, meubel of complex meubel, met gebruik van:
- `id`
- `namespace:id`

Wanneer de inventaris door alle spelers wordt gesloten, keert de opslag terug naar zijn normale variant.
