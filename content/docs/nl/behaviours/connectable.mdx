De ID's moeten aangepaste meubels zijn.

## Standaard ID-resolutie

Alle meubel-type ID's zijn optioneel. Als ze niet zijn opgegeven, zoekt de plugin automatisch naar overeenkomende meubel-ID's:

- `default` erft de meubel-ID waarop het gedrag is geconfigureerd.
- Elk ander type gebruikt `<meubelnaam>_<type>`.

Als het gedrag bijvoorbeeld is geconfigureerd op een meubel met de naam `connectable_furniture`:

### Automatische lookup trap-type

- `default` - `connectable_furniture`
- `straight` - `connectable_furniture_straight`
- `left` - `connectable_furniture_left`
- `right` - `connectable_furniture_right`
- `outer` - `connectable_furniture_outer`
- `inner` - `connectable_furniture_inner`

### Automatische lookup tafel-type

- `default` - `connectable_furniture`
- `straight` - `connectable_furniture_straight`
- `middle` - `connectable_furniture_middle`
- `border` - `connectable_furniture_border`
- `corner` - `connectable_furniture_corner`
- `end` - `connectable_furniture_end`

## Minimale configuratie

```yaml
behaviours:
  connectable:
    type: stair # of table
```

In de meeste gevallen hoef je alleen het type in te stellen; automatische ID-resolutie regelt de rest.
