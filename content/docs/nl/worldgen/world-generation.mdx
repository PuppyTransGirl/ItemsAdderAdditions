Met twee world generation-modi kun je custom furniture spawnen in nieuw gegenereerde chunks. Beide worden direct in je ItemsAdder content-YML-bestanden gedefinieerd.

| | World Populator | Surface Decorator |
|-|-----------------|-------------------|
| YAML key | `blocks_populators` | `surface_decorators` |
| Plaatsingsstijl | Vein-based (geclusterd) | Verspreid over het oppervlak |
| Dichtheidscontrole | `vein_blocks` + `chunk_veins` | `amount` |
| Surface-config key | `replaceable_blocks` | `bottom_blocks` |

Gebruik de **World Populator** voor geclusterde spawns (paddenstoelen, ertsachtige groepen). Gebruik de **Surface Decorator** voor gelijkmatig verspreide objecten aan het oppervlak (bloemen, kleine stenen).

## World Populator

Spawnt furniture in vein-achtige clusters tijdens chunkgeneratie. Gebruikt de key `blocks_populators` (de legacy key `worlds_populators` wordt ook geaccepteerd).

```yaml
info:
  namespace: my_pack

blocks_populators:
  big_mushroom:
    enabled: true
    furniture: "my_pack:big_mushroom"
    worlds:
      - world
    biomes:
      - minecraft:dark_forest
    replaceable_blocks:
      - GRASS_BLOCK
      - DIRT
    vein_blocks: 1
    chunk_veins: 3
    chunk_chance: 25.0
    min_height: 60
    max_height: 100
```

| Sleutel | Type | Standaard | Beschrijving |
|---------|------|-----------|--------------|
| `enabled` | boolean | `true` | Zet op `false` om over te slaan zonder te verwijderen. |
| `furniture` | `namespace:id` | *(vereist)* | De ItemsAdder-furniture die moet spawnen. Zie [ID's](/nl/docs/compatibility/ids). |
| `worlds` | list | *(alle)* | Wereldnamen om tot te beperken. Ondersteunt wildcardpatronen zoals `world_*`. |
| `biomes` | list | *(alle)* | Biome-ID's (bijv. `minecraft:forest`). Alle IDs moeten geldig zijn, anders wordt de entry overgeslagen. |
| `replaceable_blocks` | list | *(elk)* | Toegestane materialen voor het blok direct onder het spawnpunt. |
| `vein_blocks` | integer | `1` | Furniture-stukken per veinpoging. |
| `chunk_veins` | integer | `4` | Veinpogingen per chunk. |
| `chunk_chance` | double | `-1` | Kans per chunk van 0-100. `-1` = altijd. |
| `min_height` | integer | `60` | Minimum Y-niveau. Moet lager zijn dan `max_height`. |
| `max_height` | integer | `80` | Maximum Y-niveau. |
| `allow_liquid_placement` | boolean | `false` | Sta spawnen in water of lava toe. |

Legacy aliases worden nog geaccepteerd: `worlds_populators` voor `blocks_populators`, `amount` voor `vein_blocks`, `iterations` voor `chunk_veins` en `chance` voor `chunk_chance`.

## Surface Decorator

Verspreidt furniture over het oppervlak van nieuw gegenereerd terrein. Gebruikt de key `surface_decorators`.

```yaml
info:
  namespace: my_pack

surface_decorators:
  small_flower:
    enabled: true
    furniture: "my_pack:small_flower"
    worlds:
      - world
    biomes:
      - minecraft:plains
      - minecraft:sunflower_plains
    bottom_blocks:
      - GRASS_BLOCK
    amount: 4
    chunk_chance: 50.0
    min_height: 60
    max_height: 120
```

| Sleutel | Type | Standaard | Beschrijving |
|---------|------|-----------|--------------|
| `enabled` | boolean | `true` | Zet op `false` om over te slaan zonder te verwijderen. |
| `furniture` | `namespace:id` | *(vereist)* | De ItemsAdder-furniture die moet spawnen. Zie [ID's](/nl/docs/compatibility/ids). |
| `worlds` | list | *(alle)* | Wereldnamen om tot te beperken. Ondersteunt wildcardpatronen zoals `world_*`. |
| `biomes` | list | *(alle)* | Biome-ID's (bijv. `minecraft:plains`). Alle IDs moeten geldig zijn, anders wordt de entry overgeslagen. |
| `bottom_blocks` | list | *(elk)* | Toegestane materialen voor het blok direct onder het spawnpunt. |
| `amount` | integer | `4` | Plaatsingspogingen per chunk. |
| `chunk_chance` | double | `-1` | Kans per chunk van 0-100. `-1` = altijd. |
| `min_height` | integer | `60` | Minimum Y-niveau. Moet lager zijn dan `max_height`. |
| `max_height` | integer | `80` | Maximum Y-niveau. |
| `allow_liquid_surface` | boolean | `false` | Sta een vloeistofblok onder het spawnpunt toe als geldig oppervlak. |
| `allow_liquid_placement` | boolean | `false` | Sta spawnen in water of lava toe. |

De legacy alias `chance` wordt geaccepteerd voor `chunk_chance`.

## Opmerkingen

- Alleen nieuw gegenereerde chunks activeren spawning. Bestaande chunks worden niet beĂŻnvloed.
- De furniture moet door ItemsAdder zijn geregistreerd voordat de entry wordt geladen.
- Ongeldige biome-ID's zorgen ervoor dat de volledige entry wordt overgeslagen.
- Surface decorators in de Nether scannen omlaag van `max_height` naar `min_height`.
- Wanneer `allow_liquid_placement` is ingeschakeld, gebruikt surface decoration in normale dimensies de ocean-floor heightmap zodat onderwateroppervlakken gevonden kunnen worden.
