Twee wereldgeneratiemodi laten je aangepaste meubels spawnen in nieuw gegenereerde chunks. Beide zijn rechtstreeks gedefinieerd in je ItemsAdder-inhoud YML-bestanden.

| | Wereld-populator | Oppervlakte-decorateur |
|-|-----------------|-------------------|
| YAML-sleutel | `blocks_populators` | `surface_decorators` |
| Plaatsingsstijl | Adergebaseerd (geclusterd) | Oppervlaktespreiding |
| Dichtheidscontrole | `vein_blocks` + `chunk_veins` | `amount` |
| Oppervlakteconfiguratiesleutel | `replaceable_blocks` | `bottom_blocks` |

Gebruik de **Wereld-populator** voor geclusterde spawns (paddenstoelen, ertsachtige groepen). Gebruik de **Oppervlakte-decorateur** voor gelijkmatig verspreide oppervlakteobjecten (bloemen, kleine rotsen).

## Wereld-populator

Spawnt meubels in adergroepen tijdens chunkgeneratie. Gebruikt de sleutel `blocks_populators` (de verouderde sleutel `worlds_populators` wordt ook geaccepteerd).

```yaml
info:
  namespace: my_pack

blocks_populators:
  big_mushroom:
    enabled: true
    furniture: "my_pack:big_mushroom"
    worlds:
      - world
    biomes:
      - minecraft:dark_forest
    replaceable_blocks:
      - GRASS_BLOCK
      - DIRT
    vein_blocks: 1
    chunk_veins: 3
    chunk_chance: 25.0
    min_height: 60
    max_height: 100
```

| Sleutel | Type | Standaard | Beschrijving |
|---------|------|-----------|--------------|
| `enabled` | boolean | `true` | Stel in op `false` om over te slaan zonder te verwijderen. |
| `furniture` | `namespace:id` | *(verplicht)* | Het te spawnen ItemsAdder-meubel. |
| `worlds` | lijst | *(alle)* | Wereldnamen om tot te beperken. Ondersteunt `*` jokerteken. |
| `biomes` | lijst | *(alle)* | Bioom-ID's (bijv. `minecraft:forest`). Alle moeten geldig zijn, anders wordt de invoer overgeslagen. |
| `replaceable_blocks` | lijst | *(elk)* | Toegestane materialen voor het blok direct onder het spawnpunt. |
| `vein_blocks` | integer | `1` | Meubelstuks per aderpog ing. |
| `chunk_veins` | integer | `4` | Aderpogingen per chunk. |
| `chunk_chance` | double | `-1` | 0-100 kans per chunk. `-1` = altijd. |
| `min_height` | integer | `60` | Minimum Y-niveau. Moet minder zijn dan `max_height`. |
| `max_height` | integer | `80` | Maximum Y-niveau. |
| `allow_liquid_placement` | boolean | `false` | Sta spawnen in water of lava toe. |

## Oppervlakte-decorateur

Verspreidt meubels over het oppervlak van nieuw gegenereerd terrein. Gebruikt de sleutel `surface_decorators`.

```yaml
info:
  namespace: my_pack

surface_decorators:
  small_flower:
    enabled: true
    furniture: "my_pack:small_flower"
    worlds:
      - world
    biomes:
      - minecraft:plains
      - minecraft:sunflower_plains
    bottom_blocks:
      - GRASS_BLOCK
    amount: 4
    chunk_chance: 50.0
    min_height: 60
    max_height: 120
```

| Sleutel | Type | Standaard | Beschrijving |
|---------|------|-----------|--------------|
| `enabled` | boolean | `true` | Stel in op `false` om over te slaan zonder te verwijderen. |
| `furniture` | `namespace:id` | *(verplicht)* | Het te spawnen ItemsAdder-meubel. |
| `worlds` | lijst | *(alle)* | Wereldnamen om tot te beperken. Ondersteunt `*` jokerteken. |
| `biomes` | lijst | *(alle)* | Bioom-ID's (bijv. `minecraft:plains`). Alle moeten geldig zijn, anders wordt de invoer overgeslagen. |
| `bottom_blocks` | lijst | *(elk)* | Toegestane materialen voor het blok direct onder het spawnpunt. |
| `amount` | integer | `4` | Plaatsingspogingen per chunk. |
| `chunk_chance` | double | `-1` | 0-100 kans per chunk. `-1` = altijd. |
| `min_height` | integer | `60` | Minimum Y-niveau. Moet minder zijn dan `max_height`. |
| `max_height` | integer | `80` | Maximum Y-niveau. |
| `allow_liquid_surface` | boolean | `false` | Sta een vloeistofblok onder het spawnpunt toe als geldig oppervlak. |
| `allow_liquid_placement` | boolean | `false` | Sta spawnen in water of lava toe. |

## Opmerkingen

- Alleen nieuw gegenereerde chunks activeren spawning. Bestaande chunks worden niet beinvloed.
- Het meubel moet door ItemsAdder zijn geregistreerd voordat de invoer wordt geladen.
- Ongeldige bioom-ID's zorgen ervoor dat de volledige invoer wordt overgeslagen.
