ItemsAdderAdditions voegt een geavanceerd craftingrecepttype toe genaamd `iaa_crafting_table`. Het werkt voor zowel 3x3 recepten als 2x2 recepten in het spelerinventaris. `iaa_crafting` wordt geaccepteerd als alias.

Het vervangt of overschrijft het normale ItemsAdder `crafting_table`-recepttype niet. In plaats daarvan biedt het een ander receptsysteem voor gevallen waarbij je extra functies voor ingrediëntvalidatie nodig hebt.

Dit recepttype ondersteunt normale knutselreceptopties die al bekend zijn van ItemsAdder, en voegt ook extra functies toe zoals:

- vanilla item-/bloktags
- ItemsAdder-, MMOItems- en vanilla ingrediënten
- ingrediënthoeveelheden
- duurzaamheidsschade op ingrediënten
- vervangingsitems
- duurzaamheidsmatch negeren
- potiontype-filtering
- shapeless ingrediëntenlijsten

## Belangrijk

Recepten die deze extra functies gebruiken, moeten worden geplaatst onder:

```yaml
recipes:
  iaa_crafting_table:
```

```yaml
recipes:
  iaa_crafting:
```

Gebruik bij voorkeur `iaa_crafting_table` in nieuwe bestanden. `iaa_crafting` werkt hetzelfde en bestaat als alias.

Als je alleen normale ItemsAdder-knutselfuncties nodig hebt, kun je het gewone `crafting_table`-recepttype blijven gebruiken.

## Basis gevormd recept

```yaml
info:
  namespace: my_pack

recipes:
  iaa_crafting_table:
    wooden_scythe:
      pattern:
        - "XPP"
        - "PXS"
        - "XXS"
      ingredients:
        P: "#minecraft:planks"
        S: STICK
      result:
        item: "my_pack:wooden_scythe"
```

## Basis shapeless recept

```yaml
info:
  namespace: my_pack

recipes:
  iaa_crafting_table:
    mega_diamond:
      shapeless: true
      ingredients:
        - item: IRON_INGOT
          amount: 2
        - item: GOLD_INGOT
          amount: 2
        - item: COPPER_INGOT
          amount: 2
      result:
        item: DIAMOND
        amount: 3
```

## Structuur

```yaml
info:
  namespace: my_pack

recipes:
  iaa_crafting_table:
    my_recipe:
      enabled: true
      permission: "my_pack.recipe.use"
      pattern:
        - "ABC"
        - "DEF"
        - "GHI"
      ingredients:
        A: STICK
      result:
        item: DIAMOND
        amount: 1
```

## Ondersteunde standaard receptfuncties

`iaa_crafting_table` ondersteunt ook veelgebruikte knutselfuncties die je mogelijk al kent van ItemsAdder, zoals:

- `enabled`
- `permission`
- gevormde recepten met `pattern`
- shapeless recepten
- meerdere `pattern...`-varianten
- `potion_type` ingrediëntfiltering

## Extra ItemsAdderAdditions-functies

De belangrijkste reden om `iaa_crafting_table` te gebruiken is de aanvullende ondersteuning voor ingrediëntpredicaten:

- vanilla tags met `#minecraft:tag`
- MMOItems-ingrediënten met `mmoitems:type:id`
- per-ingrediënt `amount`
- per-ingrediënt `damage`
- per-ingrediënt `replacement`
- `ignore_durability`
- shapeless lijststijl ingrediëntformaat

## Parameters

### `enabled`

Optioneel. Standaard: `true`.

Als ingesteld op `false`, wordt het recept niet geregistreerd.

### `permission`

Optioneel.

Als ingesteld, moeten spelers deze permissie hebben om het receptresultaat te zien en te craften.

```yaml
permission: "my_pack.recipe.use"
```

### `shapeless`

Optioneel. Standaard: `false`.

Als `true`, wordt het recept behandeld als shapeless.

## Voltooiingsacties

Gebruik `on_complete` om een geluid af te spelen en/of commando's uit te voeren wanneer het recept wordt gemaakt.

```yaml
on_complete:
  sound:
    name: "minecraft:entity.player.levelup"
    source: master # optioneel, standaard: master
    volume: 1.0 # optioneel, standaard: 1.0
    pitch: 1.0 # optioneel, standaard: 1.0
  commands:
    reward:
      command: "give {player} gold_ingot 4"
      as_console: true # optioneel, standaard: false
```

`{player}` wordt vervangen door de naam van de speler die het recept maakt. Commando's worden standaard als speler uitgevoerd; stel `as_console: true` in om ze vanuit de console uit te voeren.

## Resultaat

### `result.item`

Verplicht.

Het uitvoeritem.

Ondersteunt vanilla materialen, ItemsAdder-items en MMOItems-items. Tags zijn alleen geldig voor ingrediënten, niet voor resultaten.

```yaml
result:
  item: "my_pack:custom_sword"
```

### `result.amount`

Optioneel. Standaard: `1`.

```yaml
result:
  item: DIAMOND
  amount: 3
```

## Gevormde recepten

Gevormde recepten gebruiken een `pattern` en een met tekens gesleutelde `ingredients`-sectie.

```yaml
my_recipe:
  pattern:
    - "XPP"
    - "PXS"
    - "XXS"
  ingredients:
    P: "#minecraft:planks"
    S: STICK
  result:
    item: "my_pack:wooden_scythe"
```

### Patroonopmerkingen

- Ingrediëntsleutels moeten een enkel teken zijn.
- `X` markeert lege slots. Definieer `X` niet als ingrediënt.
- Elk ander teken dat niet in `ingredients` is gedefinieerd, wordt ook als leeg behandeld.
- Meerdere patronen worden ondersteund met sleutels zoals `pattern_1`, `pattern_2`, enzovoort.

## Shapeless recepten

Shapeless recepten gebruiken:

```yaml
shapeless: true
```

en kunnen een lijst van ingrediënten gebruiken:

```yaml
my_recipe:
  shapeless: true
  ingredients:
    - item: IRON_INGOT
      amount: 2
    - item: GOLD_INGOT
      amount: 2
  result:
    item: DIAMOND
```

## Ingrediëntformaten

Ingrediënten ondersteunen meerdere formaten.

### Eenvoudige string

```yaml
ingredients:
  S: STICK
  P: "#minecraft:planks"
  C: "my_pack:custom_part"
  M: "mmoitems:material:ruby_shard"
```

### Gedetailleerd item

```yaml
ingredients:
  S:
    item: STICK
    amount: 2
    damage: 5
    replacement: STICK
    ignore_durability: true
    potion_type: minecraft:water
```

### Shapeless lijststijl

```yaml
ingredients:
  - SNOWBALL
  - item: WATER_BUCKET
    replacement: BUCKET
```

## Geavanceerde ingrediëntfuncties

### Tags

Gebruik vanilla Minecraft item- of bloktags door de waarde met `#` te prefixen. Tags matchen alleen vanilla registry-items/blokken; ze matchen geen ItemsAdder- of MMOItems-content.

```yaml
ingredients:
  P: "#minecraft:planks"
```

Dit laat elk item in die tag overeenkomen.

### `amount`

Vereist meer dan een van hetzelfde ingrediënt.

```yaml
ingredients:
  I:
    item: IRON_INGOT
    amount: 3
```

### `damage`

In plaats van het ingrediënt te verbruiken, wordt het beschadigd met het geconfigureerde bedrag.

Handig voor gereedschappen gebruikt bij craften.

```yaml
ingredients:
  H:
    item: IRON_PICKAXE
    damage: 10
```

### `replacement`

Vervang het ingrediënt door een ander item na het craften.

Handig voor emmers, flessen, mallen en soortgelijke containers.

```yaml
ingredients:
  W:
    item: WATER_BUCKET
    replacement: BUCKET
```

### `ignore_durability`

Negeer de huidige duurzaamheidswaarde bij het matchen van een item.

```yaml
ingredients:
  T:
    item: IRON_PICKAXE
    ignore_durability: true
```

### `potion_type`

Beperk een potioningrediënt tot een specifiek basis-potiontype.

```yaml
ingredients:
  P:
    item: POTION
    potion_type: minecraft:jump_boost
```

## Volledig gevormd voorbeeld

```yaml
info:
  namespace: my_pack

recipes:
  iaa_crafting_table:
    reinforced_blade:
      enabled: true
      permission: "my_pack.recipes.reinforced_blade"
      pattern:
        - "XIT"
        - "XSI"
        - "SXX"
      ingredients:
        I:
          item: IRON_INGOT
          amount: 2
        S: STICK
        T:
          item: IRON_PICKAXE
          damage: 8
          ignore_durability: true
      result:
        item: "my_pack:reinforced_blade"
        amount: 1
```

## Volledig shapeless voorbeeld

```yaml
info:
  namespace: my_pack

recipes:
  iaa_crafting_table:
    purified_bottle:
      shapeless: true
      ingredients:
        - item: POTION
          potion_type: minecraft:water
        - item: DIAMOND
          amount: 2
      result:
        item: "my_pack:purified_bottle"
```

## Ondersteunde itemreferenties

Zie [ID's](/nl/docs/compatibility/ids) voor de gedeelde ID-referentie.

Ingrediënt- en resultaatitems kunnen gebruiken:

- vanilla materiaalnamen zoals `STICK` of `DIAMOND`
- vanilla namespaced ID's zoals `minecraft:stick`
- ItemsAdder-ID's zoals `namespace:item`
- MMOItems-ID's zoals `mmoitems:type:id`

Ingrediënten kunnen daarnaast vanilla tags gebruiken zoals `#minecraft:planks` en ItemsAdderAdditions custom item-tags. Tags zijn niet geldig als resultaat of vervangingsitem.

## Opmerkingen

- `replacement` en `damage` worden toegepast tijdens het craften.
- Shapeless recepten met geavanceerde predicaten moeten het lijstformaat gebruiken.
- `iaa_crafting_table` is een aanvullend geavanceerd recepttype, geen vervanging voor ItemsAdder `crafting_table`.
- `iaa_crafting` is een alias voor `iaa_crafting_table`.
- Elke `pattern...` key wordt geladen als een shaped pattern variant. Voorbeelden: `pattern`, `pattern_2`, `pattern_alt`.
- Anonieme shapeless ingrediëntenlijsten ondersteunen maximaal 26 entries.
