Voeg steensnijderrecepten toe zonder extra receptbestanden aan te maken.

## Parameters

### `enabled`

Optioneel. Standaard: `true`. Stel dit in op `false` om het registreren van het recept over te slaan.

### `ingredient.item`

Verplicht. Het invoeritem dat in de steensnijder wordt getoond. Gebruik een vanilla materiaal, een ItemsAdder-item of een MMOItems-item.

```yaml
ingredient:
  item: STONE
```

```yaml
ingredient:
  item: my_pack:rough_marble
```

```yaml
ingredient:
  item: mmoitems:material:rough_marble
```

### `result.item`

Verplicht. Het uitvoeritem.

```yaml
result:
  item: STONE_BRICKS
```

### `result.amount`

Optioneel. Standaard: `1`. Het aantal geproduceerde items.

## Volledig voorbeeld

```yaml
info:
  namespace: my_pack

recipes:
  stonecutter:
    marble_tiles:
      ingredient:
        item: my_pack:rough_marble
      result:
        item: my_pack:marble_tiles
        amount: 2
```

## Opmerkingen

- Ondersteunt vanilla, ItemsAdder- en MMOItems-items.
- Tags zoals `#minecraft:stone_tool_materials` worden hier niet ondersteund, omdat de steensnijder één concrete invoerstack nodig heeft.
- Recept-ID's zoals `marble_tiles` zijn interne identificatoren.
- De namespace komt van `info.namespace`.
