Voeg kampvuurbereidingsrecepten toe zonder extra receptbestanden aan te maken.

## Parameters

### `enabled`

Optioneel. Standaard: `true`. Stel dit in op `false` om het registreren van het recept over te slaan.

### `ingredient.item`

Verplicht. Het invoeritem voor het recept. Gebruik een vanilla materiaal, een ItemsAdder-item of een MMOItems-item.

```yaml
ingredient:
  item: BEEF
```

```yaml
ingredient:
  item: my_pack:raw_fish_fillet
```

```yaml
ingredient:
  item: mmoitems:consumable:raw_fish
```

### `result.item`

Verplicht. Het bereide uitvoeritem.

```yaml
result:
  item: COOKED_BEEF
```

### `result.amount`

Optioneel. Standaard: `1`. Het aantal geproduceerde items.

### `cook_time`

Optioneel. Standaard: `600` ticks. Bereidingstijd. 20 ticks = 1 seconde.

### `exp`

Optioneel. Standaard: `0.0`. Ervaring verdiend bij het nemen van het bereide resultaat.

## Volledig voorbeeld

```yaml
info:
  namespace: my_pack

recipes:
  campfire_cooking:
    roasted_berry_skewer:
      ingredient:
        item: my_pack:berry_skewer_raw
      result:
        item: my_pack:berry_skewer_roasted
        amount: 1
      cook_time: 160
      exp: 0.25
```

## Opmerkingen

- Ondersteunt vanilla, ItemsAdder- en MMOItems-items.
- Tags zoals `#minecraft:fishes` worden hier niet ondersteund, omdat kampvuurrecepten één concrete invoerstack nodig hebben.
- Recept-ID's zoals `roasted_berry_skewer` zijn interne identificatoren.
- De namespace komt van `info.namespace`.
