---
title: Triggers
description: All 24 built-in trigger types for custom advancements and their conditions.
version: "1.0.9"
---

Triggers define what the player must do to complete a criterion. Each criterion has one trigger and an optional `conditions` block.

```yaml
criteria:
  <criterion_id>:
    trigger: <trigger_type>
    conditions:
      # trigger-specific options
```

An advancement can have multiple criteria. All of them must be completed before the advancement is awarded.

---

## obtain_item

Fires when the player picks up an item from the ground.

```yaml
criteria:
  get_sword:
    trigger: obtain_item
    conditions:
      items:
        - my_pack:ruby_sword
        - my_pack:iron_sword
      amount: 1
```

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `items` | list of strings | `[]` | ItemsAdder item IDs. Any match triggers the criterion. |
| `amount` | integer | `1` | Minimum stack size the player must pick up at once. |

---

## consume_item

Fires when the player finishes eating or drinking an item.

```yaml
criteria:
  eat_ruby_apple:
    trigger: consume_item
    conditions:
      item: my_pack:ruby_apple
```

| Key | Type | Required | Description |
|-----|------|----------|-------------|
| `item` | string | yes | ItemsAdder item ID. |

---

## using_item

Fires when the player right-clicks (in air or on a block) while holding a custom item in their main hand.

```yaml
criteria:
  use_staff:
    trigger: using_item
    conditions:
      item: my_pack:ruby_staff
```

| Key | Type | Required | Description |
|-----|------|----------|-------------|
| `item` | string | yes | ItemsAdder item ID. |

---

| Key | Type | Required | Description |
|-----|------|----------|-------------|
| `block` | string | yes | ItemsAdder block ID (`namespace:id`). |

## kill_entity_with_item

Fires when the player kills an entity while holding a specific custom item in their main hand.

```yaml
criteria:
  slay_zombie:
    trigger: kill_entity_with_item
    conditions:
      item: my_pack:ruby_sword
      entity_type: minecraft:zombie  # optional
```

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `item` | string | *(required)* | ItemsAdder item ID the player must be holding. |
| `entity_type` | string | *(any)* | Vanilla entity type key (e.g. `minecraft:zombie`). If omitted, any entity matches. |

---

## player_hurt_entity

Fires when the player deals damage to an entity. Unlike `kill_entity_with_item`, this fires on every hit, not just the killing blow.

```yaml
criteria:
  hit_zombie:
    trigger: player_hurt_entity
    conditions:
      item: my_pack:ruby_sword  # optional
      entity_type: minecraft:zombie  # optional
```

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `item` | string | *(any)* | ItemsAdder item ID the player must be holding. If omitted, any item (or bare hands) matches. |
| `entity_type` | string | *(any)* | Vanilla entity type key. If omitted, any entity matches. |

---

## changed_dimension

Fires when the player moves to a different dimension.

```yaml
criteria:
  enter_nether:
    trigger: changed_dimension
    conditions:
      dimension: nether  # optional
```

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `dimension` | string | *(any)* | Dimension name: `normal`, `nether`, or `the_end`. If omitted, any dimension change matches. |

---

## permission

Fires when the player joins while holding the specified permission node.

```yaml
criteria:
  has_vip:
    trigger: permission
    conditions:
      permission: "myserver.rank.vip"
```

| Key | Type | Required | Description |
|-----|------|----------|-------------|
| `permission` | string | yes | Permission node. Checked on every join; fires as soon as the player has it. |

---

## in_biome

Fires when the player crosses into a specific biome. Checked at chunk boundaries.

```yaml
criteria:
  visit_jungle:
    trigger: in_biome
    conditions:
      biome: minecraft:jungle
      world: world  # optional
```

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `biome` | string | *(required)* | Biome key as `namespace:id` (e.g. `minecraft:deep_dark`). |
| `world` | string | *(any)* | World name to restrict the check to a single world. |

---

## impossible

Never fires on its own. The criterion can only be granted via the IAA API or admin commands.

```yaml
criteria:
  manual:
    trigger: impossible
```

No `conditions` options.
