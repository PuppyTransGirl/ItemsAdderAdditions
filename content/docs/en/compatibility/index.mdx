Compatibility pages explain how ItemsAdderAdditions talks to ItemsAdder and optional plugins. Use this section when you need to know whether an integration is required, how an external plugin affects a feature, or which ID syntax a field accepts.

## Supported integrations

**Required:**

- [ItemsAdder](/en/docs/compatibility/itemsadder) - content loading, custom stacks, custom blocks/furniture, item modifier hooks, and reload events.

**Optional:**

- [MMOItems](/en/docs/compatibility/mmoitems) - `mmoitems:type:id` item references and runtime item matching.
- [ValhallaMMO & ValhallaTrinkets](/en/docs/compatibility/valhalla) - Valhalla metadata written to ItemsAdder item stacks.
- [WorldGuard](/en/docs/compatibility/worldguard) - region flags for supported behaviours, custom paintings, and actions.
- [CoreProtect](/en/docs/compatibility/coreprotect) - storage open interaction logging.
- [PlaceholderAPI](/en/docs/compatibility/placeholderapi) - placeholder parsing in supported text fields.
- [MythicMobs](/en/docs/compatibility/mythicmobs) - the `mythic_mobs_skill` action.
- [AntiGriefLib](/en/docs/compatibility/antigrieflib) - build-permission filtering for `replace_biome`.

## ID syntax

For item, block, furniture, recipe, tag, and namespaced ID formats, see [IDs](/en/docs/compatibility/ids).
