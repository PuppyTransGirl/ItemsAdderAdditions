ItemsAdderAdditions resolves item, block, furniture, recipe, and tag references through a shared compatibility layer. The exact supported formats depend on the feature because some options must create a concrete item stack while others only need to match existing content.

## Server and plugin support

ItemsAdder is required. Any ItemsAdder v4 version can be used, but the latest version is recommended. PlaceholderAPI, CoreProtect, MMOItems, ValhallaMMO, and ValhallaTrinkets are optional integrations used only by features that reference them.

Features backed by newer Minecraft registries are version-gated:

- Creative inventory integration requires a 1.21.5+ compatible NMS adapter.
- Custom paintings require a 1.21.5+ compatible NMS adapter.
- Generic components require a 1.21.3+ compatible NMS adapter.

On unsupported versions, version-gated features are skipped and the plugin logs a warning.

## Supported ID formats

| Content | Format | Example |
|---------|--------|---------|
| ItemsAdder item | `namespace:id` or bare `id` inside the current `info.namespace` | `my_pack:ruby_sword` |
| ItemsAdder block | `namespace:id` or bare `id` inside the current `info.namespace` | `my_pack:ruby_ore` |
| ItemsAdder furniture | `namespace:id` or bare `id` inside the current `info.namespace` | `my_pack:ruby_lamp` |
| MMOItems item | `mmoitems:type:id` | `mmoitems:sword:flame_blade` |
| Vanilla item | `minecraft:id`, Bukkit material name, or bare Minecraft key | `minecraft:diamond`, `DIAMOND`, `diamond` |
| Vanilla block | `minecraft:id`, Bukkit material name, or bare Minecraft key | `minecraft:oak_log`, `OAK_LOG`, `oak_log` |
| Vanilla item/block tag | `#minecraft:tag` | `#minecraft:planks`, `#minecraft:logs` |
| ItemsAdderAdditions custom tag | `#namespace:tag` or bare `#tag` in the current namespace | `#my_pack:ruby_tools` |

IDs are normalized to lowercase internally. For readability, prefer fully qualified IDs such as `minecraft:diamond` and `my_pack:ruby_sword`.

## Custom tags

ItemsAdderAdditions custom tags are defined in a top-level `tags` section and can group item, block, furniture, or recipe IDs.

```yaml
info:
  namespace: my_pack

tags:
  ruby_tools:
    type: item
    values:
      - ruby_pickaxe
      - ruby_axe
```

Custom tags are resolved before vanilla tags. See [Custom Tags](/en/docs/custom-tags) for the full schema, nesting rules, and supported feature list.

## Vanilla tags

Vanilla tags are still supported where the feature uses Bukkit item or block tag matching.

```yaml
# Any vanilla plank item
item: "#minecraft:planks"

# Any vanilla log block
block: "#minecraft:logs"
```

Vanilla tags only match vanilla registry entries. They do not match ItemsAdder content or MMOItems content.

`iaa_crafting_table` ingredient tags are vanilla Bukkit item/block tags only. Crafting ingredients do not resolve ItemsAdderAdditions custom tags.

## Concrete item outputs

Some options must create a specific item stack. Tags are not valid in those places because a tag can match multiple values.

Use concrete IDs for:

- recipe results
- campfire, stonecutter, and brewing results
- `toast.icon`
- `clear_item.item`
- `replace_item.item`
- storage or action options that need to create one item stack

## MMOItems

MMOItems support is optional. If MMOItems is installed and enabled, use:

```yaml
item: mmoitems:sword:flame_blade
```

The plugin detects MMOItems stacks by their MMOItems type and ID. If MMOItems is not loaded, `mmoitems:type:id` references cannot be resolved and will not match.

## Rotated ItemsAdder blocks and furniture

When matching ItemsAdder blocks or furniture at runtime, directional suffixes are treated as the same base content:

```text
my_pack:chair_north -> my_pack:chair
my_pack:chair_south -> my_pack:chair
my_pack:chair_east  -> my_pack:chair
my_pack:chair_west  -> my_pack:chair
my_pack:chair_up    -> my_pack:chair
my_pack:chair_down  -> my_pack:chair
```

This makes advancement triggers and runtime checks work with rotated variants without having to list every direction.

## Recipe examples

Crafting ingredients can use vanilla tags, ItemsAdder items, MMOItems items, and vanilla items:

```yaml
ingredients:
  P: "#minecraft:planks"
  M: "mmoitems:material:ruby_shard"
  I: "my_pack:reinforced_ingot"
result:
  item: "my_pack:ruby_handle"
```

Campfire, stonecutter, and brewing recipes use concrete item references:

```yaml
recipes:
  campfire_cooking:
    cooked_mmo_fish:
      ingredient:
        item: mmoitems:consumable:raw_fish
      result:
        item: mmoitems:consumable:cooked_fish
```

## Advancement examples

```yaml
criteria:
  chop_any_log:
    trigger: break_block
    conditions:
      block: "#minecraft:logs"
```

```yaml
criteria:
  obtain_custom_tool:
    trigger: obtain_item
    conditions:
      items:
        - "#my_pack:ruby_tools"
        - mmoitems:sword:flame_blade
        - minecraft:diamond_sword
```
