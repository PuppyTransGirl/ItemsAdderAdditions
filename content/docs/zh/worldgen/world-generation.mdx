两种世界生成模式可让你在新生成的区块中生成自定义家具。两者都直接在你的 ItemsAdder 内容 YML 文件中定义

|        | 世界填充器                         | 地表装饰器                |
|--------|-------------------------------|----------------------|
| YAML 键 | `blocks_populators`           | `surface_decorators` |
| 放置风格   | 基于矿脉（簇状）                      | 地表散布                 |
| 密度控制   | `vein_blocks` + `chunk_veins` | `amount`             |
| 表面配置键  | `replaceable_blocks`          | `bottom_blocks`      |

使用**世界填充器**可进行簇状生成（蘑菇、类似矿石的群组）。使用**地表装饰器**可进行均匀散布的地表物体生成（花朵、小石头）

## 世界填充器

在区块生成期间以矿脉式簇状生成家具。使用 `blocks_populators` 键（也接受旧版键 `worlds_populators`）

```yaml
info:
  namespace: my_pack

blocks_populators:
  big_mushroom:
    enabled: true
    furniture: "my_pack:big_mushroom"
    worlds:
      - world
    biomes:
      - minecraft:dark_forest
    replaceable_blocks:
      - GRASS_BLOCK
      - DIRT
    vein_blocks: 1
    chunk_veins: 3
    chunk_chance: 25.0
    min_height: 60
    max_height: 100
```

| 键                        | 类型        | 默认值     | 描述                                                  |
|--------------------------|-----------|---------|-----------------------------------------------------|
| `enabled`                | 布尔值       | `true`  | 设置为 `false` 可跳过而不移除                                 |
| `furniture`              | `命名空间:id` | *（必填）*  | 要生成的 ItemsAdder 家具                                  |
| `worlds`                 | 列表        | *（全部）*  | 要限定的世界名称。支持 `*` 通配符                                 |
| `biomes`                 | 列表        | *（全部）*  | 生物群系 ID（例如 `minecraft:forest`）。所有 ID 必须有效，否则该条目将被跳过 |
| `replaceable_blocks`     | 列表        | *（任意）*  | 生成点正下方方块允许的材质                                       |
| `vein_blocks`            | 整数        | `1`     | 每次矿脉尝试放置的家具数量                                       |
| `chunk_veins`            | 整数        | `4`     | 每个区块的矿脉尝试次数                                         |
| `chunk_chance`           | 双精度浮点数    | `-1`    | 每个区块的概率 0-100。`-1` = 始终生成                           |
| `min_height`             | 整数        | `60`    | 最低 Y 坐标。必须小于 `max_height`                           |
| `max_height`             | 整数        | `80`    | 最高 Y 坐标                                             |
| `allow_liquid_placement` | 布尔值       | `false` | 允许在水中或熔岩中生成                                         |

## 地表装饰器

在新生成的地形表面散布家具。使用 `surface_decorators` 键

```yaml
info:
  namespace: my_pack

surface_decorators:
  small_flower:
    enabled: true
    furniture: "my_pack:small_flower"
    worlds:
      - world
    biomes:
      - minecraft:plains
      - minecraft:sunflower_plains
    bottom_blocks:
      - GRASS_BLOCK
    amount: 4
    chunk_chance: 50.0
    min_height: 60
    max_height: 120
```

| 键                        | 类型        | 默认值     | 描述                                                  |
|--------------------------|-----------|---------|-----------------------------------------------------|
| `enabled`                | 布尔值       | `true`  | 设置为 `false` 可跳过而不移除                                 |
| `furniture`              | `命名空间:id` | *（必填）*  | 要生成的 ItemsAdder 家具                                  |
| `worlds`                 | 列表        | *（全部）*  | 要限定的世界名称。支持 `*` 通配符                                 |
| `biomes`                 | 列表        | *（全部）*  | 生物群系 ID（例如 `minecraft:plains`）。所有 ID 必须有效，否则该条目将被跳过 |
| `bottom_blocks`          | 列表        | *（任意）*  | 生成点正下方方块允许的材质                                       |
| `amount`                 | 整数        | `4`     | 每个区块的放置尝试次数                                         |
| `chunk_chance`           | 双精度浮点数    | `-1`    | 每个区块的概率 0-100。`-1` = 始终生成                           |
| `min_height`             | 整数        | `60`    | 最低 Y 坐标。必须小于 `max_height`                           |
| `max_height`             | 整数        | `80`    | 最高 Y 坐标                                             |
| `allow_liquid_surface`   | 布尔值       | `false` | 允许将生成点下方的液体方块视为有效表面                                 |
| `allow_liquid_placement` | 布尔值       | `false` | 允许在水中或熔岩中生成                                         |

## 备注

- 仅新生成的区块会触发生成。现有区块不受影响
- 家具必须在加载该条目之前已由 ItemsAdder 注册
- 无效的生物群系 ID 会导致整个条目被跳过
