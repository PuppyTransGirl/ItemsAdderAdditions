自定义标签可以把内容 ID 分组，并在支持的 ItemsAdderAdditions 功能中复用。它们从 ItemsAdder 内容 YAML 顶层的 `tags` 节加载。

## 基础结构

```yaml
info:
  namespace: my_pack

tags:
  ruby_tools:
    type: item
    values:
      - ruby_pickaxe
      - ruby_axe
      - mmoitems:tool:ruby_hammer

  decorative_blocks:
    type: block
    values:
      - ruby_lamp
      - minecraft:amethyst_block

  trading_stations:
    type: furniture
    values:
      - alchemy_table
      - forge_table

  ruby_recipes:
    type: recipe
    values:
      - ruby_sword
      - ruby_pickaxe
```

同一 namespace 内使用 `#tag_id`，跨 namespace 使用 `#namespace:tag_id`。

## 类型

| 类型 | 别名 | 可包含的值 |
|------|------|------------|
| `item` | `items` | ItemsAdder 物品、原版物品、MMOItems 物品和嵌套物品标签。 |
| `block` | `blocks` | ItemsAdder 方块、原版方块和嵌套方块标签。 |
| `furniture` | `furnitures` | ItemsAdder 家具 ID 和嵌套家具标签。 |
| `recipe` | `recipes` | 配方 ID 和嵌套配方标签。 |

如果省略 `type`，会按物品标签处理。

## 嵌套标签

```yaml
tags:
  ruby_tools:
    type: item
    values:
      - ruby_pickaxe
      - ruby_axe

  mining_tools:
    type: item
    values:
      - "#ruby_tools"
      - minecraft:iron_pickaxe
      - minecraft:diamond_pickaxe
```

嵌套标签必须是相同类型。循环引用会被检测并忽略。

## 可用位置

自定义标签可用于运行时 ID 匹配路径，包括：

- 进度的物品、方块、家具和配方条件
- 玩家谓词的物品、装备、槽位和方块检查
- `stackable` 行为的 `items`

`iaa_crafting_table` 原料不同：原料标签只使用 Bukkit 原版物品/方块标签，不解析 ItemsAdderAdditions 自定义标签。

## 完整示例

```yaml
info:
  namespace: my_pack

tags:
  ruby_tools:
    type: item
    values:
      - ruby_pickaxe
      - ruby_axe

items:
  charged_ruby:
    display_name: "<red>充能红宝石"
    behaviours:
      stackable:
        blocks:
          - charged_ruby_block
        items:
          - "#my_pack:ruby_tools"
```

## 重载

自定义标签会在动作、行为、配方、进度和世界生成加载前准备。修改标签后执行 `/iareload`。

## 常见错误

- 混用标签类型。家具标签不能包含物品标签。
- 重复定义相同 `type` 和标签 ID。第一次定义会生效。
- 在必须创建具体物品的位置使用标签，例如配方结果、`toast.icon`、`clear_item.item` 或 `replace_item.item`。
