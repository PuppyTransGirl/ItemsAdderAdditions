---
title: Empilable
description: Permettre aux blocs personnalisés ItemsAdder de s’empiler les uns sur les autres dans une séquence définie. Prend en charge les piles simples, complexes et multicouches.
version: "1.0.0"
---

<Tabs items={['Simple', 'Complexe', 'Complet']}>
    <Tab value="Simple">
        ```yaml
        behaviours:
          stackable:
            - my_block_2
            - my_block_3
            - my_block_4
        ```
    </Tab>
    <Tab value="Complexe">
        ```yaml
        behaviours:
          stackable:
            blocks:
              - my_block_2
              - my_block_3
              - my_block_4
            items: # optionnel, par défaut l’objet sur lequel ce comportement est défini
              - my_item
              - BONE_MEAL
            sound: # optionnel
              name: entity.villager.ambient # ou un son personnalisé
              volume: 1.0 # optionnel, défaut : 1.0
              pitch: 1.0 # optionnel, défaut : 1.0
              category: MASTER # optionnel, défaut : MASTER
            decrement_amount: 2 # optionnel, défaut : 1
        ```

        Pour `name`, utilisez une valeur dans [Sounds.json](https://minecraft.wiki/w/Sounds.json/Java_Edition_values)
        ou un son personnalisé avec `namespace:value`.
    </Tab>
    <Tab value="Complet">
        ```yaml
        behaviours:
          stackable:
            first_step:
              block: my_block_2
              items: # optionnel, par défaut l’objet sur lequel ce comportement est défini
                - my_item
                - BONE_MEAL
              sound: # optionnel
                name: entity.villager.ambient # ou un son personnalisé
                volume: 1.0 # optionnel, défaut : 1.0
                pitch: 1.0 # optionnel, défaut : 1.0
                category: MASTER # optionnel, défaut : MASTER
              decrement_amount: 2 # optionnel, défaut : 1
            another_step:
              ...
        ```

        Pour `name`, utilisez une valeur dans [Sounds.json](https://minecraft.wiki/w/Sounds.json/Java_Edition_values)
        ou un son personnalisé avec `namespace:value`.
    </Tab>
</Tabs>

Les ID doivent être des blocs personnalisés. Types pris en charge : `REAL`, `REAL_NOTE`, `REAL_TRANSPARENT` et `REAL_WIRE`. Vous pouvez ajouter autant d'étapes que nécessaire.
