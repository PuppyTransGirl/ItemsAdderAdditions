ItemsAdderAdditions résout les références d’objets, de blocs, de meubles, de recettes et de tags via une couche de compatibilité commune. Les formats exactement pris en charge dépendent de la fonctionnalité, car certaines options doivent créer un objet concret tandis que d’autres n’ont qu’à correspondre à du contenu existant.

## Prise en charge serveur et plugins

ItemsAdder est requis. N’importe quelle version v4 d’ItemsAdder peut être utilisée, mais la dernière version est recommandée. PlaceholderAPI, CoreProtect, MMOItems, ValhallaMMO et ValhallaTrinkets sont des intégrations optionnelles utilisées uniquement par les fonctionnalités qui y font référence.

Les fonctionnalités reposant sur des registres Minecraft plus récents sont limitées par version :

- L’intégration de l’inventaire créatif nécessite un adaptateur NMS compatible 1.21.5+.
- Les peintures personnalisées nécessitent un adaptateur NMS compatible 1.21.5+.
- Les composants génériques nécessitent un adaptateur NMS compatible 1.21.3+.

Sur les versions non prises en charge, les fonctionnalités limitées par version sont ignorées et le plugin enregistre un avertissement.

## Formats d’IDs pris en charge

| Contenu | Format | Exemple |
|---------|--------|---------|
| Objet ItemsAdder | `namespace:id` ou `id` seul dans le `info.namespace` actuel | `my_pack:ruby_sword` |
| Bloc ItemsAdder | `namespace:id` ou `id` seul dans le `info.namespace` actuel | `my_pack:ruby_ore` |
| Meuble ItemsAdder | `namespace:id` ou `id` seul dans le `info.namespace` actuel | `my_pack:ruby_lamp` |
| Objet MMOItems | `mmoitems:type:id` | `mmoitems:sword:flame_blade` |
| Objet vanilla | `minecraft:id`, nom de matériau Bukkit ou clé Minecraft seule | `minecraft:diamond`, `DIAMOND`, `diamond` |
| Bloc vanilla | `minecraft:id`, nom de matériau Bukkit ou clé Minecraft seule | `minecraft:oak_log`, `OAK_LOG`, `oak_log` |
| Tag vanilla d’objets/blocs | `#minecraft:tag` | `#minecraft:planks`, `#minecraft:logs` |
| Tag personnalisé ItemsAdderAdditions | `#namespace:tag` ou `#tag` dans le namespace actuel | `#my_pack:ruby_tools` |

Les IDs sont normalisés en minuscules en interne. Pour une configuration plus lisible, privilégiez les IDs complets comme `minecraft:diamond` et `my_pack:ruby_sword`.

## Tags personnalisés

Les tags personnalisés d’ItemsAdderAdditions sont définis dans une section `tags` au niveau racine et peuvent regrouper des IDs d’objets, de blocs, de meubles ou de recettes.

```yaml
info:
  namespace: my_pack

tags:
  ruby_tools:
    type: item
    values:
      - ruby_pickaxe
      - ruby_axe
```

Les tags personnalisés sont résolus avant les tags vanilla. Consultez [Tags personnalisés](/fr/docs/custom-tags) pour le schéma complet, les règles d’imbrication et la liste des fonctionnalités prises en charge.

## Tags vanilla

Les tags vanilla restent pris en charge là où la fonctionnalité utilise la correspondance par tags Bukkit d’objets ou de blocs.

```yaml
# N’importe quelle planche vanilla
item: "#minecraft:planks"

# N’importe quel tronc vanilla
block: "#minecraft:logs"
```

Les tags vanilla ne correspondent qu’aux entrées du registre vanilla. Ils ne correspondent pas au contenu ItemsAdder ni au contenu MMOItems.

Les tags d’ingrédients de `iaa_crafting_table` sont uniquement des tags Bukkit vanilla d’objets/blocs. Les ingrédients de fabrication ne résolvent pas les tags personnalisés d’ItemsAdderAdditions.

## Sorties d’objets concrets

Certaines options doivent créer un objet précis. Les tags n’y sont pas valides car un tag peut correspondre à plusieurs valeurs.

Utilisez des IDs concrets pour :

- les résultats de recette
- les résultats de feu de camp, de tailleur de pierre et d’alambic
- `toast.icon`
- `clear_item.item`
- `replace_item.item`
- les options de stockage ou d’action qui doivent créer un objet

## MMOItems

Le support de MMOItems est optionnel. Si MMOItems est installé et activé, utilisez :

```yaml
item: mmoitems:sword:flame_blade
```

Le plugin détecte les objets MMOItems grâce à leur type et leur ID MMOItems. Si MMOItems n’est pas chargé, les références `mmoitems:type:id` ne peuvent pas être résolues et ne correspondent à rien.

## Blocs et meubles ItemsAdder tournés

Lors des vérifications runtime, les suffixes directionnels des blocs et meubles ItemsAdder sont traités comme le même contenu de base :

```text
my_pack:chair_north -> my_pack:chair
my_pack:chair_south -> my_pack:chair
my_pack:chair_east  -> my_pack:chair
my_pack:chair_west  -> my_pack:chair
my_pack:chair_up    -> my_pack:chair
my_pack:chair_down  -> my_pack:chair
```

Cela permet aux déclencheurs de progrès et aux vérifications runtime de fonctionner avec les variantes tournées sans devoir lister chaque direction.

## Exemples de recettes

Les ingrédients de fabrication peuvent utiliser des tags vanilla, des objets ItemsAdder, des objets MMOItems et des objets vanilla :

```yaml
ingredients:
  P: "#minecraft:planks"
  M: "mmoitems:material:ruby_shard"
  I: "my_pack:reinforced_ingot"
result:
  item: "my_pack:ruby_handle"
```

Les recettes de feu de camp, de tailleur de pierre et d’alambic utilisent des références d’objets concrets :

```yaml
recipes:
  campfire_cooking:
    cooked_mmo_fish:
      ingredient:
        item: mmoitems:consumable:raw_fish
      result:
        item: mmoitems:consumable:cooked_fish
```

## Exemples de progrès

```yaml
criteria:
  chop_any_log:
    trigger: break_block
    conditions:
      block: "#minecraft:logs"
```

```yaml
criteria:
  obtain_custom_tool:
    trigger: obtain_item
    conditions:
      items:
        - "#my_pack:ruby_tools"
        - mmoitems:sword:flame_blade
        - minecraft:diamond_sword
```
