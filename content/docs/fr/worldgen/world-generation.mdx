---
title: Génération du monde
description: Faire apparaître des meubles ItemsAdder personnalisés pendant la génération des chunks.

Deux modes de génération du monde permettent de faire apparaître des meubles personnalisés dans les chunks nouvellement générés. Les deux se définissent directement dans vos fichiers YML de contenu ItemsAdder.

| | World Populator | Surface Decorator |
|-|-----------------|-------------------|
| Clé YAML | `blocks_populators` | `surface_decorators` |
| Style de placement | Par veines (groupé) | Dispersion en surface |
| Contrôle de densité | `vein_blocks` + `chunk_veins` | `amount` |
| Clé de configuration de surface | `replaceable_blocks` | `bottom_blocks` |

Utilisez le **World Populator** pour les apparitions groupées (champignons, groupes de type minerai). Utilisez le **Surface Decorator** pour les objets de surface répartis régulièrement (fleurs, petits rochers).

## World Populator

Fait apparaître des meubles en groupes de type veine pendant la génération des chunks. Utilise la clé `blocks_populators` (la clé historique `worlds_populators` est aussi acceptée).

```yaml
info:
  namespace: my_pack

blocks_populators:
  big_mushroom:
    enabled: true
    furniture: "my_pack:big_mushroom"
    worlds:
      - world
    biomes:
      - minecraft:dark_forest
    replaceable_blocks:
      - GRASS_BLOCK
      - DIRT
    vein_blocks: 1
    chunk_veins: 3
    chunk_chance: 25.0
    min_height: 60
    max_height: 100
```

| Clé | Type | Défaut | Description |
|-----|------|--------|-------------|
| `enabled` | boolean | `true` | Définissez sur `false` pour ignorer sans supprimer. |
| `furniture` | `namespace:id` | *(requis)* | Le meuble ItemsAdder à faire apparaître. Voir [IDs](/fr/docs/compatibility/ids). |
| `worlds` | list | *(tous)* | Noms des mondes à cibler. Prend en charge les motifs wildcard comme `world_*`. |
| `biomes` | list | *(tous)* | IDs de biomes (ex. `minecraft:forest`). Tous doivent être valides ou l'entrée est ignorée. |
| `replaceable_blocks` | list | *(n'importe lequel)* | Matériaux autorisés pour le bloc directement sous le point d'apparition. |
| `vein_blocks` | integer | `1` | Nombre de meubles par tentative de veine. |
| `chunk_veins` | integer | `4` | Tentatives de veine par chunk. |
| `chunk_chance` | double | `-1` | Chance de 0 à 100 par chunk. `-1` = toujours. |
| `min_height` | integer | `60` | Niveau Y minimal. Doit être inférieur à `max_height`. |
| `max_height` | integer | `80` | Niveau Y maximal. |
| `allow_liquid_placement` | boolean | `false` | Autorise l'apparition dans l'eau ou la lave. |

Les alias historiques restent acceptés : `worlds_populators` pour `blocks_populators`, `amount` pour `vein_blocks`, `iterations` pour `chunk_veins` et `chance` pour `chunk_chance`.

## Surface Decorator

Répartit des meubles sur la surface du terrain nouvellement généré. Utilise la clé `surface_decorators`.

```yaml
info:
  namespace: my_pack

surface_decorators:
  small_flower:
    enabled: true
    furniture: "my_pack:small_flower"
    worlds:
      - world
    biomes:
      - minecraft:plains
      - minecraft:sunflower_plains
    bottom_blocks:
      - GRASS_BLOCK
    amount: 4
    chunk_chance: 50.0
    min_height: 60
    max_height: 120
```

| Clé | Type | Défaut | Description |
|-----|------|--------|-------------|
| `enabled` | boolean | `true` | Définissez sur `false` pour ignorer sans supprimer. |
| `furniture` | `namespace:id` | *(requis)* | Le meuble ItemsAdder à faire apparaître. Voir [IDs](/fr/docs/compatibility/ids). |
| `worlds` | list | *(tous)* | Noms des mondes à cibler. Prend en charge les motifs wildcard comme `world_*`. |
| `biomes` | list | *(tous)* | IDs de biomes (ex. `minecraft:plains`). Tous doivent être valides ou l'entrée est ignorée. |
| `bottom_blocks` | list | *(n'importe lequel)* | Matériaux autorisés pour le bloc directement sous le point d'apparition. |
| `amount` | integer | `4` | Tentatives de placement par chunk. |
| `chunk_chance` | double | `-1` | Chance de 0 à 100 par chunk. `-1` = toujours. |
| `min_height` | integer | `60` | Niveau Y minimal. Doit être inférieur à `max_height`. |
| `max_height` | integer | `80` | Niveau Y maximal. |
| `allow_liquid_surface` | boolean | `false` | Autorise un bloc liquide sous le point d'apparition comme surface valide. |
| `allow_liquid_placement` | boolean | `false` | Autorise l'apparition dans l'eau ou la lave. |

L'alias historique `chance` est accepté pour `chunk_chance`.

## Notes

- Seuls les chunks nouvellement générés déclenchent l'apparition. Les chunks existants ne sont pas affectés.
- Le meuble doit être enregistré par ItemsAdder avant le chargement de l'entrée.
- Les IDs de biomes invalides font ignorer toute l'entrée.
- Les surface decorators dans le Nether scannent vers le bas de `max_height` à `min_height`.
- Lorsque `allow_liquid_placement` est activé, la décoration de surface en dimension normale utilise la heightmap du fond océanique afin de trouver les surfaces sous l'eau.
