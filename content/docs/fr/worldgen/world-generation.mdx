title: Génération du monde
description: Faire apparaître des meubles ItemsAdder personnalisés lors de la génération des chunks.

Deux modes de génération du monde permettent de faire apparaître des meubles personnalisés dans les chunks nouvellement générés. Les deux sont définis directement dans vos fichiers YML de contenu ItemsAdder.

| | Populateur de monde | Décorateur de surface |
|-|---------------------|----------------------|
| Clé YAML | `blocks_populators` | `surface_decorators` |
| Style de placement | Basé sur les veines (groupé) | Dispersion en surface |
| Contrôle de densité | `vein_blocks` + `chunk_veins` | `amount` |
| Clé de config de surface | `replaceable_blocks` | `bottom_blocks` |

Utilisez le **Populateur de monde** pour les apparitions groupées (champignons, groupes semblables aux minerais). Utilisez le **Décorateur de surface** pour les objets de surface régulièrement dispersés (fleurs, petits rochers).

## Populateur de monde

Fait apparaître des meubles en grappes de type veine lors de la génération des chunks. Utilise la clé `blocks_populators` (la clé legacy `worlds_populators` est également acceptée).

```yaml
info:
  namespace: my_pack

blocks_populators:
  big_mushroom:
    enabled: true
    furniture: "my_pack:big_mushroom"
    worlds:
      - world
    biomes:
      - minecraft:dark_forest
    replaceable_blocks:
      - GRASS_BLOCK
      - DIRT
    vein_blocks: 1
    chunk_veins: 3
    chunk_chance: 25.0
    min_height: 60
    max_height: 100
```

| Clé | Type | Défaut | Description |
|-----|------|--------|-------------|
| `enabled` | booléen | `true` | Définir à `false` pour ignorer sans supprimer. |
| `furniture` | `namespace:id` | *(obligatoire)* | Le meuble ItemsAdder à faire apparaître. |
| `worlds` | liste | *(tous)* | Noms de monde à restreindre. Supporte le joker `*`. |
| `biomes` | liste | *(tous)* | IDs de biome (ex. `minecraft:forest`). Tous doivent être valides sinon l'entrée est ignorée. |
| `replaceable_blocks` | liste | *(tous)* | Matériaux autorisés pour le bloc directement sous le point d'apparition. |
| `vein_blocks` | entier | `1` | Pièces de meuble par tentative de veine. |
| `chunk_veins` | entier | `4` | Tentatives de veine par chunk. |
| `chunk_chance` | double | `-1` | Chance 0-100 par chunk. `-1` = toujours. |
| `min_height` | entier | `60` | Niveau Y minimum. Doit être inférieur à `max_height`. |
| `max_height` | entier | `80` | Niveau Y maximum. |
| `allow_liquid_placement` | booléen | `false` | Autoriser l'apparition dans l'eau ou la lave. |

## Décorateur de surface

Disperse des meubles sur la surface du terrain nouvellement généré. Utilise la clé `surface_decorators`.

```yaml
info:
  namespace: my_pack

surface_decorators:
  small_flower:
    enabled: true
    furniture: "my_pack:small_flower"
    worlds:
      - world
    biomes:
      - minecraft:plains
      - minecraft:sunflower_plains
    bottom_blocks:
      - GRASS_BLOCK
    amount: 4
    chunk_chance: 50.0
    min_height: 60
    max_height: 120
```

| Clé | Type | Défaut | Description |
|-----|------|--------|-------------|
| `enabled` | booléen | `true` | Définir à `false` pour ignorer sans supprimer. |
| `furniture` | `namespace:id` | *(obligatoire)* | Le meuble ItemsAdder à faire apparaître. |
| `worlds` | liste | *(tous)* | Noms de monde à restreindre. Supporte le joker `*`. |
| `biomes` | liste | *(tous)* | IDs de biome (ex. `minecraft:plains`). Tous doivent être valides sinon l'entrée est ignorée. |
| `bottom_blocks` | liste | *(tous)* | Matériaux autorisés pour le bloc directement sous le point d'apparition. |
| `amount` | entier | `4` | Tentatives de placement par chunk. |
| `chunk_chance` | double | `-1` | Chance 0-100 par chunk. `-1` = toujours. |
| `min_height` | entier | `60` | Niveau Y minimum. Doit être inférieur à `max_height`. |
| `max_height` | entier | `80` | Niveau Y maximum. |
| `allow_liquid_surface` | booléen | `false` | Autoriser un bloc liquide sous le point d'apparition comme surface valide. |
| `allow_liquid_placement` | booléen | `false` | Autoriser l'apparition dans l'eau ou la lave. |

## Notes

- Seuls les chunks nouvellement générés déclenchent l'apparition. Les chunks existants ne sont pas affectés.
- Le meuble doit être enregistré par ItemsAdder avant que l'entrée soit chargée.
- Les IDs de biome invalides provoquent l'ignorance de toute l'entrée.
