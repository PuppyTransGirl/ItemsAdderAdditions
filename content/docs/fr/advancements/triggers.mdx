---
title: Déclencheurs
description: Référence de tous les types de déclencheurs intégrés disponibles pour les progrès ItemsAdder personnalisés, avec les conditions et les exemples YAML de chacun.
version: "1.0.9"
---

Les déclencheurs définissent ce que le joueur doit faire pour terminer un critère. Chaque critère a un déclencheur et un bloc `conditions` optionnel.

```yaml
criteria:
  <criterion_id>:
    trigger: <trigger_type>
    conditions:
      # options spécifiques au déclencheur
```

Un progrès peut avoir plusieurs critères. Tous doivent être terminés avant que le progrès soit accordé.

---

## obtain_item

Se déclenche lorsque le joueur ramasse un objet du sol.

```yaml
criteria:
  get_sword:
    trigger: obtain_item
    conditions:
      items:
        - my_pack:ruby_sword
        - my_pack:iron_sword
      amount: 1
```

| Clé | Type | Par défaut | Description |
|-----|------|---------|-------------|
| `items` | liste de chaînes | `[]` | ID d’objets (IA, MMOItems ou vanille). Toute correspondance déclenche le critère. |
| `amount` | entier | `1` | Taille minimale de pile que le joueur doit ramasser à la fois. |

---

## consume_item

Se déclenche lorsque le joueur termine de manger ou de boire un objet.

```yaml
criteria:
  eat_ruby_apple:
    trigger: consume_item
    conditions:
      item: my_pack:ruby_apple
```

| Clé | Type | Requis | Description |
|-----|------|----------|-------------|
| `item` | chaîne | Oui | ID d’objet (IA, MMOItems ou vanille). |

---

| Clé | Type | Requis | Description |
|-----|------|----------|-------------|

## kill_entity_with_item

Se déclenche lorsque le joueur tue une entité tout en tenant un objet spécifique dans sa main principale.

```yaml
criteria:
  slay_zombie:
    trigger: kill_entity_with_item
    conditions:
      item: my_pack:ruby_sword
      entity_type: minecraft:zombie  # optionnel
```

| Clé | Type | Par défaut | Description |
|-----|------|---------|-------------|
| `item` | chaîne | *(requis)* | ID d’objet (IA, MMOItems ou vanille) que le joueur doit tenir. |
| `entity_type` | chaîne | *(n’importe quel)* | Clé de type d'entité vanille (par ex. `minecraft:zombie`). Si omis, toute entité correspondra. |

---

## player_hurt_entity

Se déclenche lorsque le joueur inflige des dégâts à une entité. Contrairement à `kill_entity_with_item`, cela se déclenche à chaque coup, pas seulement le coup fatal.

```yaml
criteria:
  hit_zombie:
    trigger: player_hurt_entity
    conditions:
      item: my_pack:ruby_sword  # optionnel
      entity_type: minecraft:zombie  # optionnel
```

| Clé | Type | Par défaut | Description |
|-----|------|---------|-------------|
| `item` | chaîne | *(n’importe quel)* | ID d’objet que le joueur doit tenir. Si omis, tout objet (ou mains nues) correspondra. |
| `entity_type` | chaîne | *(n’importe quel)* | Clé de type d'entité vanille. Si omis, toute entité correspondra. |

## permission

Se déclenche lorsque le joueur se connecte tout en ayant le nœud de permission spécifié.

```yaml

    trigger: permission
    conditions:
      permission: "myserver.rank.vip"
```

| Clé | Type | Requis | Description |
|-----|------|----------|-------------|
| `permission` | chaîne | oui | Nœud de permission. Vérifié à chaque connexion ; se déclenche dès que le joueur l'a. |

## in_biome

Se déclenche lorsque le joueur entre dans un biome spécifique. Vérifié aux limites des chunks.

```yaml
criteria:
  visit_jungle:
    trigger: in_biome
    conditions:
      biome: minecraft:jungle
      world: world  # optionnel
```

| Clé | Type | Par défaut | Description |
|-----|------|---------|-------------|
| `biome` | chaîne | *(requis)* | Clé de biome comme `namespace:id` (par ex. `minecraft:deep_dark`). |
| `world` | chaîne | *(n’importe quel)* | Nom du monde pour limiter la vérification à un seul monde. |

---

## impossible

Ne se déclenche jamais d'elle-même. Le critère ne peut être accordé que via l'API IAA ou les commandes d'administrateur.

```yaml
criteria:
  manual:
    trigger: impossible
```

Pas d'options `conditions`.
