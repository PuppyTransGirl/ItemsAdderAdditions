Chaque action prend en charge un ensemble de paramètres partagés. Ils vous permettent de conditionner les actions à des permissions, d'ajouter des délais et de contrôler à qui l’action s'applique. Vous pouvez en combiner autant que vous voulez sur une même action.

## Paramètres

### `permission`

Si défini, l’action ne se déclenche que si le joueur possède ce nœud de permission. Les joueurs sans la permission sont ignorés silencieusement.

```yaml
actionbar:
  text: "<green>Bonjour !"
  permission: "my_pack.greet"
```

### `delay`

Attendre ce nombre de ticks avant d'exécuter l’action. 20 ticks = 1 seconde. Par défaut : `0` (immédiat).

```yaml
message:
  text: "<gold>Chargement..."
  delay: 40 # 2 secondes après le déclencheur
```

### `target`

Contrôle à qui l’action s'applique. Par défaut : `self`.

| Valeur             | Description                                                                                         |
|--------------------|-----------------------------------------------------------------------------------------------------|
| `self`             | Le joueur qui a déclenché l'événement *(par défaut)*                                                |
| `other`            | L'entité avec laquelle le joueur interagit (ex. celle qui est attaquée)                             |
| `all`              | Le joueur déclencheur **et** l'entité cible                                                         |
| `radius`           | Toutes les entités dans le rayon X/Y/Z configuré autour du point d'interaction - nécessite `target_radius` |
| `in_sight`         | L'entité que le joueur regarde - nécessite `target_in_sight_distance`                               |

```yaml
message:
  text: "<red>Vous avez touché quelqu’un !"
  target: other
```

### `target_radius`

Utilisé uniquement avec `target: radius`. Définit le rayon X/Y/Z en blocs autour de la cible, du bloc d'événement ou du joueur déclencheur.

```yaml
actionbar:
  text: "<yellow>Effet de zone !"
  target: radius
  target_radius: 10
```

### `target_in_sight_distance`

Utilisé uniquement avec `target: in_sight`. Le nombre maximum de blocs à parcourir depuis les yeux du joueur lors de la recherche d’une entité. Si aucune entité n'est trouvée dans cette portée, l’action est entièrement ignorée.

```yaml
ignite:
  duration: 100
  target: in_sight
  target_in_sight_distance: 10
```

## Placement des déclencheurs

Les actions se déclarent dans les sections `events` des objets, blocs, meubles ou meubles complexes ItemsAdder.

```yaml
info:
  namespace: my_pack

items:
  ruby_staff:
    events:
      interact:
        right:
          message:
            text: "<green>Right-clicked"

  ruby_lamp:
    events:
      placed_furniture:
        interact:
          message:
            text: "<yellow>Furniture clicked"
```

Les actions `placed_block` peuvent être imbriquées sous `events.placed_block`. Les actions de meubles et meubles complexes sont chargées depuis `events.placed_furniture`.

## Noms de déclencheurs pris en charge

Les déclencheurs d'objets courants incluent :

```text
interact, interact_mainhand, interact_offhand, block_break, attack, kill,
drop, pickup, held, held_offhand, unheld, unheld_offhand, item_break,
eat, drink, bow_shot, gun_shot, gun_no_ammo, gun_reload, item_throw,
item_hit_ground, item_hit_entity, book_write, book_read, fishing_start,
fishing_caught, fishing_failed, fishing_cancel, fishing_bite,
fishing_in_ground, bucket_empty, bucket_fill
```

Les déclencheurs de blocs incluent `interact`, `break` et les déclencheurs `placed_block` imbriqués. Les déclencheurs de meubles utilisent les mêmes noms de style objet sous `placed_furniture`. Les meubles complexes prennent actuellement en charge `placed_furniture.interact`.

`interact`, `interact_mainhand` et `interact_offhand` peuvent utiliser des sections d'arguments comme `right`, `left`, `right_shift` et `left_shift`. Les actions wildcard sous le déclencheur sont aussi exécutées après la correspondance exacte de l'argument.

## Cooldowns et exigences de stats

Lorsqu'une action est déclenchée depuis un objet ItemsAdder tenu en main, ItemsAdderAdditions vérifie les cooldowns et exigences de stats configurés par ItemsAdder pour cet objet avant d'exécuter l'action. Si le joueur ne remplit pas ces exigences, l'action est ignorée.

## Exemple complet

```yaml
my_item:
  events:
    interact:
      right:
        actionbar:
          text: "<aqua>Clic droit !"
          permission: "some.permission" # Seuls les joueurs avec cette permission déclencheront ceci
          delay: 10 # Délai d’une demi-seconde
          target: self # Appliquer uniquement au cliqueur
```
